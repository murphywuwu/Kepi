《客批》架构总览图

Next.js · App Router · Zustand · Canvas 2D · 纯 TS 引擎 · AI 代理 · 本机存档

Runtime

  • 单页主战场，状态切换完成整局体验。
  • 外层 Server Component，主体 Client Component。
  • Canvas 负责棋盘与战斗，React 负责外围 UI。

Domain

  • 纯 TS reducer 引擎是唯一真相。
  • engine / data / store / lib 分层明确。
  • Zod 负责输入、配置、API 返回校验。

Delivery

  • AI 走 app/api 代理，弱网自动降级。
  • localStorage 只存设置与可选快照。
  • Vitest + Playwright 兜住核心流程。
