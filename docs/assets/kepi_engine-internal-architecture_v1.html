《客批》引擎内部架构图

pure TS reducer engine · state machine · battle / economy / shop / progression · Zod · tests

Entry

  • UI actions enter through Zustand.
  • Zod guards the payloads first.
  • Reducer returns a fresh snapshot.

Core

  • stateMachine decides the current scene.
  • battle / economy / shop / progression are split.
  • deterministic outputs keep tests easy.

Data

  • TS const modules are the source of truth for configs.
  • shared types feed UI, engine, API, and tests.
  • no runtime DB, no duplicated truth.

Fallbacks

  • AI can drop to local letter pool at any time.
  • localStorage only stores local snapshots and settings.
  • debug page helps diagnose rule drift fast.
