《客批》数据流与状态机图

UI → Zustand → pure TS reducer → Canvas / API / localStorage · scene machine included

Input

  • 玩家操作先进入 Zustand。
  • Zod 做入口校验，防止坏数据流入引擎。
  • 引擎只吃合法动作。

Loop

  • prep / battle / settlement / ending 四态流转。
  • battle 是固定 tick，结果可复现。
  • settlement 才真正推进胜负与资源。

Output

  • snapshot 回到 store，再给 UI 和 Canvas 渲染。
  • localStorage 只存本机数据。
  • ending 时才可能触发 AI 代理。
