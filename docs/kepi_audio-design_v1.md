# 《客批》音效与音乐设计文档 v1

> 目的：作为游戏音频专家，基于 [PRD V3.1](kepi_PRD_V3.1.md)、[关卡交互设计 v1](kepi_level-interaction-design_v1.md)、[美术风格设定 v1](kepi_art-style-design_v1.md) 与客家侨批史实，定稿《客批》的**音乐主题体系、BGM 分幕设计、SFX 规范与动态混音总线**，作为 BGM 重做与音效迭代的唯一依据。
> 依据：V3.1 七关情绪曲线、客家五声/八音/山歌腔、Web Audio API 程序化合成栈（`src/lib/audio/`）

## 0. 一句话定位

> **玩家听到的不是"背景音乐"，而是"一封正在被风浪吞没、又被一双沾满盐霜的手死死攥住的家书"。**

《客批》的全部声音只有一个母题——**归乡**。所有 BGM 与 SFX 都必须服务于"百年客家侨民在归乡路上护信、典信、抢信"的情感曲线，而非普通的自走棋功能音。

## 1. 设计总纲

### 1.1 文化根：客家声音的三重基因

| 基因 | 来源 | 在游戏中的声音化身 |
|---|---|---|
| **客家山歌腔** | 梅县/永定山歌，高亢悠长、句尾常带下滑拖腔 | 旋律主题的"拖尾音"，BGM 中长音弱化的 sine 滑落 |
| **客家八音** | 丝竹与打击并置（唢呐、竹笛、椰胡、扬琴、锣、钹、木鱼、扁鼓） | 合成器音色选型：triangle≈竹笛，sine+detune≈椰胡，噪声 hit≈木鱼/扁鼓 |
| **客家五声音阶** | 宫商角徵羽（C-D-E-G-A），无半音、天然温婉带乡愁 | 全部旋律/和声的音阶约束；do-sol 五度是"家"的根音对位 |

> **铁律**：旋律/和声**禁止使用 fa(B)与 ti(F)**（半音），五声内做文章。唯一例外是"罪恶感/典当"主题，引入一个降序的半音经过音（如 bB 经由 A 滑向 G），制造"走调的心痛"。

### 1.2 情感四大支柱（决定四种音乐情绪态）

| 支柱 | 情绪态 | 听觉特征 | 出现场景 |
|---|---|---|---|
| **乡愁** | 温婉、悠长、空 | 长延音 sine pad + 五声拨弦，留白多 | 路线推进、篝火、结局归乡 |
| **罪恶感**（典当烧信） | 沉、走调、心坠 | 低频 thud + 半音下滑 + 静默 | 典当行仪式 |
| **守护/紧张**（死保水客） | 心跳、压迫、鼓点加密 | 低频脉冲 + 木鱼加密 + ducking | 战斗、水客危机 |
| **风浪/终局** | 混沌、吞噬、最终一束光 | 噪声海浪 + 全频段压制 + 一线五声高光 | 第7关、风浪抓信 |

### 1.3 三条不可违背的音频红线

1. **断网可玩、零素材依赖**：除客家话朗读外，全部 BGM/SFX 由 Web Audio API 实时合成（现状已达成，重做 BGM 保持此约束）。
2. **音量统一受控**：`masterGain`（设置音量）→ `bgmGain`(0.55x) + `sfxGain`，单一总线（现状已达成）。
3. **BGM 必须有情绪弧线**：禁止"一段 loop 走全场"。BGM 须随 7 关推进与场景节点动态变化（**这是本次重做的核心目标**）。

## 2. 现状诊断：当前 BGM 为什么不好

当前 `src/lib/audio/bgm.ts` 的问题（即用户"不喜欢"的根因）：

| 问题 | 现状 | 后果 |
|---|---|---|
| **无主题动机** | 随机从 `SCALE[]` 抓音，没有可记忆的旋律线 | 听过即忘，没有"客批的音乐"标识 |
| **无情绪弧线** | 一段 72BPM 静态 loop 走全场，备战=战斗=篝火 | 与 7 关推进脱节，听感平、腻 |
| **无文化签名** | 纯西方 sine/triangle，没有客家山歌腔/八音质感 | 像任意生成器 demo，不像客家游戏 |
| **pad 太闷** | C2+G2 双 sine drone，低频糊成一团 | 与五声旋律打架，听感 muddy |
| **打击无意义** | 每 4 步一记木鱼 tick，机械、与剧情无关 | 像节拍器而非"归乡脚步" |
| **ducking 粗暴** | 战斗直接压到 0.32 倍一刀切 | 情绪无层次 |

**重做方向**：用**主题动机（Leitmotif）+ 分幕音乐图（Music Map）+ 客家音色质感**，重建 BGM。

## 3. 音乐主题动机体系（Leitmotif）

四个贯穿全剧的主题动机，在 BGM/SFX 中以变奏形式反复出现，形成"音乐叙事"。

### 3.1 主题 A：「乡愁」（归乡母题）

- **音型**：`A4 → C5 → G4 → E4 →（长延音）G4`（五声，do-mi-sol 围绕，句尾下滑拖腔=客家山歌腔）
- **音色**：sine 主旋律 + triangle 八度低垫（椰胡感）
- **情绪**：温婉、牵挂、回望
- **出现**：开场、路线推进间奏、归乡结局。战斗中作为"水客安全"的暗线。

### 3.2 主题 B：「家·土楼」（守护母题）

- **音型**：`C4 → G4 → C5 → G4`（纯五度跳进，do-sol 是"家"的根音对位）
- **音色**：triangle 和弦 + 低频木质 thud（夯土感）
- **情绪**：稳固、庇护、可被修复
- **出现**：土楼修复里程碑（33/66/99）、战斗开战前土楼 Buff 提示。tier 越高，主题 B 越完整。

### 3.3 主题 C：「罪恶·典当」（失母题）

- **音型**：`A3 → bA3 → G3`（半音下滑，**全剧唯一允许的半音**）+ 低频 thud
- **音色**：低 sine + 闷噪声
- **情绪**：心坠、走调、不可逆
- **出现**：典当仪式。信烧尽时，主题 C 沉入静默，留 0.8s 真空。

### 3.4 主题 D：「风浪」（终局母题）

- **音型**：五声在高音区碎片化、被噪声海浪吞没，最后一线 `C6 → G6` 高光
- **音色**：低通噪声海浪 + 极弱高音 sine 碎片
- **情绪**：混沌、吞噬、最终一束光
- **出现**：第7关、风浪抓信、救信结局。

> **变奏原则**：主题不原样重复，按情绪态变奏（转调/改音色/改节奏/改配器密度）。听者应在潜意识中感知"同一段乡愁在不同关变得更沉或更亮"。

## 4. BGM 分幕音乐图（核心：随 7 关推进）

这是本次重做的核心。BGM 不再是单 loop，而是**一个状态机驱动的音乐层系统**，随 `game phase` 与 `journeyIndex` 切换。

### 4.1 音乐层架构（分层而非单轨）

| 层 | 角色 | 音色 | 触发 |
|---|---|---|---|
| **L0 Pad 层** | 持续氛围（乡愁底色/危机底色） | sine + 低通噪声 | 始终在，按情绪态切音色与音域 |
| **L1 旋律层** | 主题动机变奏 | triangle/sine（椰胡/竹笛感） | 路线推进、结局；战斗中弱化或静默 |
| **L2 节奏层** | 木鱼/扁鼓/脚步 | 滤波噪声 hit | 战斗时加密；篝火时极疏或无 |
| **L3 主题高光层** | 关键时刻主题动机完整奏响 | 多 osc 和弦 | 里程碑/结局 |

### 4.2 七关 + 节点 BGM 情绪态图

| 关卡 | 场景 | 情绪态 | L0 Pad | L1 旋律 | L2 节奏 | 主题高光 |
|---|---|---|---|---|---|---|
| 1 南洋余波 | 篝火·启程 | 乡愁·暖 | C2+G2 sine，暖 | 主题A完整，慢 | 无 | — |
| 1→2 路线推进 | 卷轴 | 乡愁·行 | 同上+行路低噪 | 主题A变奏，加椰胡感 | 极疏木鱼=脚步 | — |
| 2 海禁余波 | 战斗 | 守护·压 | C2 单 sine，压低 | 主题A碎片化 | 木鱼加密 + 低频脉冲 | 主题B（tier提示） |
| 3 关隘盘查 | 战斗 | 守护·排队 | +路引纸噪声 | 主题A更碎 | 扁鼓规整点 | 主题B |
| 4 契约束缚 | 战斗 | 罪恶·缠 | +锁链噪声暗线 | 主题C动机潜伏 | 链条 sss 声 | 主题B |
| 4 典当行 | 典当仪式 | **罪恶·心坠** | 压至静默 | **主题C完整** | 印章 thud | 主题C |
| 5 饿虎山 | 战斗 | 守护·冷 | 降调 pad（转 G 小调感） | 主题A变暗 | 山风低噪 + 鼓 | 主题B |
| 5 篝火夜话 | 篝火 | 乡愁·诉 | 暖 pad 回 | 主题A慢奏 | 火噼啪 | — |
| 6 红头船 | 战斗 | 风浪·急 | +海浪暗线 | 主题A急促变奏 | 鼓加密 | 主题B/D 混 |
| 7 械斗火 | 终局战斗 | 风浪·终 | 主题D pad | 主题A与D对位 | 火噪声 + 鼓 | 主题B（tier99） |
| 7 风浪抓信 | 结局演出 | **风浪·光** | 海浪主导 | 主题A碎片→C6高光 | 无 | **主题A完整（归乡）** |

### 4.3 关键过渡的设计

- **进入战斗**：`duckBgm(0.32)` → L1 旋律静默、L2 节奏加密、L0 pad 转危机音色。保留主题动机在低声部隐约可闻，让玩家潜意识知道"还在那条归乡路上"。
- **水客濒危**：BGM 进一步 duck + 注入心跳低频脉冲（与 `playWaterGuestHeartbeatSfx` 同源），制造"音乐也被掐住呼吸"的窒息感。
- **土楼里程碑**：触发**主题 B 高光层**一次完整奏响（覆盖 ducking，短时 `restoreBgm` + 主题 B 上扬），完成后回落。
- **典当仪式**：`stopBgm` 短暂静默 → 主题 C 主题音 + 信烧噪声 → 金币落地 → 0.8s 真空 → BGM 以更暗的变奏重启（"典当过的路，BGM 也回不到原来的亮"）。
- **结局归乡**：`stopBgm` → 海浪（主题 D pad）→ 抓信瞬间主题 A 高光完整奏响 → 客家话朗读。

## 5. SFX 设计规范

### 5.1 合成原语清单（`synth.ts` 现状 + 扩展建议）

| 原语 | 现状 | 用途 |
|---|---|---|
| `playTone` | ✓ ADSR 振荡器 | 单音/钟铃 |
| `playNoise` | ✓ 滤波噪声 | 打击/风/海浪 |
| `playPluck` | ✓ 三角拨弦 | 旋律点/购买 |
| `playChord` | ✓ 和弦 | swell/里程碑 |
| `playGlide` | ✓ 频率滑音 | 技能/攻速 |
| `playSequence` | ✓ 序列 | 琶音/胜利 |
| `noteToFreq` / `buildMelodySequence` | ✓ 纯函数 | 可单测 |

> **建议扩展**（BGM 重做时新增）：`playPadSwell`（多 sine 缓起 pad）、`playDetunedPair`（双 osc 微失谐=椰胡颤音感）、`scheduleLayer`（前瞻调度的分层音乐，复用现有 lookahead 机制）。

### 5.2 SFX 事件表（按场景，含主题动机关联）

> 标注「●主题X」表示该 SFX 引用对应动机片段，强化音乐叙事统一性。

#### 商店 / 备战
| 事件 | 现状 SFX | 设计要点 |
|---|---|---|
| 购买 | `playBuySfx` 上行 C5→G5 | 五声上行，明亮；可微调为「●主题A起音」 |
| 刷新 | `playRefreshSfx` 噪声+拨弦 | 纸张翻动感（bandpass 噪声） |
| 升人口 | `playPopulationSfx` CEG 三和弦 | ●主题B（家）的稳固和弦 |
| 升星 | `playStarUpSfx` 五声四连 | 加一点高频 shimmer，与土楼免死区分 |
| 卖出 | `playSellSfx` 下行 | 下行但不过分悲，仅"轻退还" |

#### 战斗
| 事件 | 现状 SFX | 设计要点 |
|---|---|---|
| 攻击/受击 | `playHitSfx(damage)` 噪声 burst | 节流 80ms；按伤害变调（已实现）；建议区分"我方受击更闷、敌方受击更脆" |
| 技能 | `playSkillSfx` 上滑 | 普通技能用；**土楼免死用 ●主题B 高光**而非通用技能音 |
| 落叶归根 | `playLeafFallSfx` 五声拨串 | **大招 stinger**：前置 ●主题A 上行琶音 + 全屏 shimmer，强化"归根"叙事 |
| 水客心跳 | `playWaterGuestHeartbeatSfx` | 与 BGM L2 节奏层同步，形成"心跳=BGM 鼓点"的窒息 |
| 水客阵亡 | `playWaterGuestDeathSfx` | 低长音 + 0.5s 静默 + 客家话叹息（朗读兜底） |

#### 典当行（核心情感记忆点）
| 事件 | 现状 SFX | 设计要点 |
|---|---|---|
| 盖章 | `playPawnStampSfx` 闷击+低 thud | ●主题C 半音动机；印章 thud 要"沉到胃" |
| 信燃烧 | （待补） | 滤波噪声渐弱 + 纸灰 sss；伴随 BGM 静默 |
| 金币落地 | `playPawnGoldSfx` 双 ping | 清脆金属，**与沉闷盖章形成情感反差**（钱来了，心空了） |
| 透支未来（泣血） | （待补） | ●主题C 完整 + 低频颤音；"卖未来"的眩晕感 |

#### 节点 / 开场
| 事件 | 现状 SFX | 设计要点 |
|---|---|---|
| 抓乡音符 | `playOpeningBuffSfx` warm chord | ●主题A 上行 swell；三种 Buff 可微变音色 |
| 篝火选择 | `playCampfireChoiceSfx` 拨弦 | 暖单音；可选随 scenario 变调（南洋暖/险路冷） |
| 路线节点切换 | （建议补） | ●主题A 变奏片段 + 行路低噪，作为"卷轴滚动"的听觉锚 |
| 土楼修复里程碑 | `playRepairHomeSfx` + `tulouMilestone` | ●主题B 完整高光（33水井叮咚→66砌墙鼓→99灯火 shimmer），三档递进 |

#### 结算 / 结局
| 事件 | 现状 SFX | 设计要点 |
|---|---|---|
| 胜利 | `playWinSfx` 五声上行 | ●主题A 完整上行收束 |
| 失败 | `playLoseSfx` 下行 | ●主题C 动机；"信被吹回"的失落 |
| 收信 | `playCollectLetterSfx` 钟铃 | 柔钟，●主题A 句尾拖腔 |
| 结局海浪 | `playWaveSfx` 低通噪声 | ●主题D；与 BGM 结局层融合 |
| 抓住信（终局） | （建议补） | ●主题A 高光 C6→G6 + shimmer，全剧最高光一音 |
| 客家话朗读 | `playFile` | 真实录音，BGM 全停让位 |

## 6. 动态混音总线

```
                        settings.volume
                              │
                         masterGain
                  ┌───────────┴───────────┐
              bgmGain(0.55x)            sfxGain
                  │                         │
        ┌─────────┼─────────┐         (各 SFX 临时 osc)
      L0 pad   L1 旋律   L2 节奏   L3 高光
        │         │         │         │
        └─────────┴────┬────┴─────────┘
                       │
                  duckBgm(ratio) ← 战斗/水客危机
```

### 6.1 层级音量约定（相对 masterGain）

| 层 | 默认 | 战斗 duck | 水客濒危 | 典当静默 |
|---|---|---|---|---|
| L0 Pad | 0.18 | 0.10 | 0.06 | 0.00 |
| L1 旋律 | 0.14 | 0.00（静默） | 0.00 | 0.00 |
| L2 节奏 | 0.08 | 0.12（加密） | 0.14（心跳同步） | 0.00 |
| L3 高光 | 0.00 | 触发时 0.22 | — | — |

### 6.2 Ducking 分级（替代现状一刀切）

| 场景 | duck ratio | 行为 |
|---|---|---|
| 普通战斗 | 0.45 | L1 静默，L2 加密，L0 保留 |
| 水客濒危 | 0.30 | 进一步压，注入心跳脉冲 |
| 结局海浪 | 0.00（stop） | BGM 全停，让位海浪+朗读 |
| 土楼里程碑 | 1.00（restore） | 短时回满 + 主题B高光 |

## 7. 语音与朗读（文件依赖，断网降级）

- **真实客家话侨批朗读**：`playFile(src)`，唯一依赖文件，缺失静默。
- 文件路径见 [素材计划 v3 §2.12C](kepi_assets-and-media-plan_v3.md)（叶和仁/林阿发/张明德）。
- 朗读触发时 BGM 必 `stopBgm`/duck 至 0，朗读结束后视结局决定是否回 BGM。
- **水客阵亡客家话叹息**：同样走 `playFile`，缺失则仅留低长音。

## 11. V2 音色重做记录（2026-07-07）

> 背景：用户反馈「音效还是很难听」。在**断网可玩、零素材依赖**硬约束下，无法引入 Suno/Udio 等文件化 AI 音乐，
> 故 V2 重做聚焦于**程序化合成音色本身的质感与文化辨识度**（即真正能在游戏里发声的部分）。

### 11.1 根因

| 旧症 | 根因 | V2 修复 |
|---|---|---|
| 旋律无句法 | `scheduleMelody` 每 tick 随机敲一个动机音、忽略 `beats`，轮廓被压平 | **动机成句**：按每个音的 `beats` 连续奏出乐句，短语首音按密度掷骰，跳过则歇一小节 |
| 全程干瘪 | 无混响/空间，像合成器 demo | `context.ts` 总线加**算法脉冲响应混响**（山洞感，零文件）+ 暖色低通（4200Hz 缓坡） |
| pad 糊闷 | C2+G2 双 sine 低频打架 | 改 `triangle` 主体 + 高八度 sine「气声」泛音（极低增益）打开闷感 |
| 节奏机械 | 40ms 噪声 tick 像节拍器 | 非战斗用**木鱼**（带通瞬态 + 高频正弦），战斗用**战鼓**（膜鸣下滑 + 皮面噪声）；终关强拍加**大锣** |
| SFX 薄脆 | 受击/胜利/里程碑缺厚度 | 受击加膜鸣 thump；胜利/升星/收信接入**大锣**与**木鱼**；敌人攻击加战鼓/木鱼重量感 |
| 音乐硬切 | 场景切换直接重配 | `setBgmScene` 走 `crossfadeBgm`（先压到 0.32 再 0.42s 回满） |

### 11.2 新增合成原语（`synth.ts`）

| 原语 | 音色 | 用途 |
|---|---|---|
| `playWoodblock(freq)` | 木鱼/梆子：高频正弦 + 带通噪声瞬态 | BGM 节奏层（行路/篝火/典当）、购买/刷新/收信/土楼护盾 |
| `playTaiko()` | 战鼓：膜鸣正弦下滑（180→64Hz）+ 皮面低通噪声 | 战斗节奏层、受击、失败、家园修缮、敌人重击 |
| `playGong(fundamental)` | 大锣：6 个非谐泛音簇 + 长衰减 + 颤音闪烁 | 胜利收束、升星 shimmer、终关强拍、土楼免死 |
| `scheduleVoice(freq, atTime)` | 椰胡/山歌嗓音：sine+triangle 双失谐 + 颤音 LFO + 起音微滑 | BGM 旋律层与主题高光（替代单薄 sine） |

### 11.3 设计原则坚守

- **五声铁律不变**：旋律/和声仍仅用宫商角徵羽，`MOTIFS` 四个动机字节级不变（测试 `synth.test.ts` 全过）。
- **零素材依赖不变**：混响为算法脉冲响应，全部音色实时合成，断网可玩。
- **调用契约不变**：`bgm.ts` / `sfx.ts` 公共 API 与签名零改动，所有调用点无需修改。

## 12. V3 文件化配乐（Music Cog，2026-07-07）

> 在 V2 程序化音色基础上，引入 **Music Cog（CellCog）生成的成品 MP3** 作为各场景主配乐，
> 让游戏拥有真实编曲层次与民族乐器质感；同时保留程序化合成作为**缺素材自动回退**，
> 严格延续「断网可玩、缺素材不影响运行」硬约束。

### 12.1 架构

- 新增 `src/lib/audio/bgmFiles.ts`：文件化 BGM 播放层。单个 `<audio loop>` 经
  `MediaElementSource → fileBgmGain(0.6) → {masterGain 干, bgmReverbSend→convolver 湿}`，
  复用全局音量与山洞混响；场景切换走 `fileBgmGain` 交叉淡入。
- `bgm.ts` 的 `setBgmScene` 改为**文件优先**：命中 `FILE_TRACKS[scene]` 且文件存在则播 MP3
  并停程序化调度；加载失败（`error` 事件）记入 `failedFileUrls` 并静默重启程序化兜底。
- `duckBgm/restoreBgm` 同步 duck 文件层（危机时音乐下沉）。
- `context.ts` 新增 `fileBgmGain` 节点并接入主/混响总线。
- 结局：`useEndingAudio` 改调 `stopProceduralBgm()`，仅停合成、保留成品「结局底噪」作朗读底床。

### 12.2 曲目清单（scripts/gen_bgm.py，待 CELLCOG_API_KEY 生成）

文件名遵循素材规范 `public/audio/bgm/kepi_bgm_<scene>.mp3`，统一客家五声音阶 + 民族乐器：

| scene | 时长 | 情绪 / 配器 |
|---|---|---|
| `menu` | 90s | 乡愁慢板：椰胡独奏 + 古筝 pad + 远笛 + 木鱼心跳 |
| `route` | 90s | 行路备战：木鱼步伐 + 低胡根音 + 古筝琶音 + 笛变奏 |
| `campfire` | 80s | 篝火夜话：古筝 + 低胡呢喃 + 木鱼如薪裂 + 山歌气声 |
| `pawn_shop` | 75s | 典当压抑：低胡颤弓 + 闷锣 + 木裂噪 + 稀疏古筝长混响 |
| `battle` | 75s | 战斗压迫：战鼓驱动 + 低胡根音 + 古筝断奏 + 笛线 |
| `battle_final` | 90s | 终关归乡：大鼓 + 英雄椰胡奏乡愁全动机 + 古筝跑动 + 大锣强拍 |
| `ending` | 120s | 结局底噪：海潮气声 pad + 椰胡哼鸣乡愁 + 极疏笛，低动态留白 |

### 12.3 运行

```bash
export CELLCOG_API_KEY="sk_..."      # 必需（Music Cog 密钥）
```
