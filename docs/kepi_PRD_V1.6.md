## 1. 需求概述

### 1.1 需求背景

腾讯云黑客松（AI CAN DO IT · 游戏极限开发挑战赛）参赛项目，融合**赛题一公益 + 赛题二文化**双题。三人团队、限时约 3–4 天开发。

**业务痛点 / 切入点**：
- 传统国风自走棋（云顶之弈、金铲铲）靠历史名人的"全民数值共识"撑起棋子体系，客家文化没有这样一套全民英雄名录——但这恰是切入点：**客家棋子等级不靠武力，靠宗族社会地位**（族长 > 风水先生 > 乡贤 > 教书先生 > 守卫 > 农夫），这套逻辑客家人有共鸣、外人也秒懂，是国风自走棋做不出的差异化。
- 客家侨批（海外番客寄回老家的"银信合一"家书）2013 年入选《世界记忆名录》，承载近代客家人离乡背井、终身难返的家国乡愁，是极强且少被游戏化的文化母题。
- 公益与文化在市面产品里常是"贴皮"。本项目要让文化与公益**等于机制**，而非贴在机制上。

### 1.2 需求目标

- **体验目标（升华版，可一句话引用）**：让玩家在一局 AI 自走棋里，亲历"过番—寄批—难归"的客家百年乡愁——你打的每一仗，都是为了换一封能回家的信；你攒下的每一分善意，都是当年侨民"省下口粮也要反哺家乡"的回响。最终你亲手接住一封真实的侨批（客批），听它用客家话念出来——那一刻，你让一个客家人的牵挂，回了家。
  > 注：本句偏"作品介绍/路演旁白"调性，做内部产品文档时可压缩为一句话验收目标（如"玩家在一局自走棋内完成攒信→归乡的完整情感弧线，并通过手势接信达成情感峰值"）。完整版作品介绍开场文案见 §2.4。
- **赛事目标（双题融合 + AI 落地 + 公益支线显性化）**：
  - **文化题**：围龙屋=棋盘、宗族=羁绊、侨批=胜利目标，文化结构本身即玩法机制。
  - **公益题**：公益不贴皮——做成一条**贯穿全局、看得见的实体支线**（水客收信攒桑梓值→乡贤修家园→土楼肉眼可见地从破败到修复），并在叙事上接续真实的客家侨民反哺史与腾讯公益（详见 §1.6）。
  - **AI 落地**：P0 两处真实 AI——**AI 生成"数字客批"**（沿途攒的信按真实侨批语气+本局战况动态生成，每局不同）+ **AI 对手/动态难度**；P1 候选见 §6.13（AI 难度旁白官、AI 生成公益事件文案）。
- **设计原则目标**：让文化与公益**等于机制**，而非贴在机制上。


### 1.3 需求范围

**本次包含（P0 必做）**：
- 自走棋核心循环：招募、升星合成、布阵、自动战斗、经济
- 客家角色棋子体系（双轴：武力值 / 公益值）
- **公益角色独立后勤位**：水客/乡贤开局常驻后方，不占战斗人口、不抢战力（解"公益双人陷阱"，见 §5.1.3A）
- 客批收集 → **归乡票显化** → 归乡通关条件
- **公益实体支线**：水客收信→桑梓值→乡贤修家园→**土楼三阶段视觉**（公益线记忆点本体）
- "遗忘军团"敌人池（6 个历史阻力拟人化敌人）
- 围屋棋盘（圆形格、上下分敌我 + 后勤区）
- 1 套主力羁绊（宗族）；迁徙留P1
- 完整战斗数值 + 寨子存续度（生存）/ 家园修复值（温情）双线逻辑
- 公益事件取舍（简版）
- 结局手势交互过场（赢/输两版，共用资产）+ 真实侨批 + 客家话朗读
- **AI 落地（两处）**：① AI 生成"数字客批"（沿途信件按真实侨批语气+本局战况动态生成）；② AI 对手 / 动态难度

**本次可选（P1，明天评估工期后定，不挤占 P0）**：
- **公益成就池**（落法二）：把公益事件升级为对应史实五层级的事件池（修祠堂/集资建乡校/捐建卫生院/国难筹赈…），每次投入点亮一个"反哺成就"，结局汇总展示"你这一生为家乡做的事"。**红线：成就仅为 flavor 标签，只换文案/点亮图标，不产生任何独立数值循环，结局汇总只展示不计分**（守方向A）。
- **AI 落点候选（详见 §6.13，附工作量，明天讨论）**：AI 难度旁白官、AI 生成公益/文化事件文案、客家 flavor / 山歌。
- 风水先生"增益格"在圆盘上的简化实现（见 §6.3 备注）。

**本次不包含（防蔓延）**：
- ❌ 客批做成独立收集/合成/交易玩法（坚决走方向A，客批仅作胜利计数器）
- ❌ 水客远航运信的风险取舍循环（会长出第二套数值系统）
- ❌ 擂茶娘 / 酿豆腐爷等器物角色（与客批主线不在一个点上，砍）
- ❌ 围龙屋向心三圈站位机制（视觉不利、复杂，砍）
- ❌ 跨角色进化升星（教书先生→乡贤→风水，违反自走棋规则，砍）
- ❌ 多人对战、联网、账号系统、家园重建养成长线

### 1.4 上下游依赖

- **前端**：自走棋棋盘与战斗渲染、商店/羁绊 UI、结局手势交互（MediaPipe Hands + 鼠标/触摸降级）
- **后端 / 全栈**：战斗结算逻辑、经济逻辑、AI 对手/动态难度技术选型、AI 内容生成（P1）
- **第三方**：手势识别库（MediaPipe Hands）；AI 接口（对手/内容生成）

- **赛务前置**：赛区报名（华南/港澳）、demo 提交形式（可玩 build / 视频）——deadline 级，尽快定

### 1.6 公益线设计依据（史实 × 腾讯公益 × 阿嬷情书 × 客批）四列咬合

> 本节是公益线"不贴皮"的论证核心，给评委一眼看清：我们的公益不是口号，而是**有史实血缘、接腾讯生态、呼应文化母题、并落成具体玩法**的四重咬合。

**史实脉络（见附件 §11.7）**：客家侨民的反哺不是单一"寄钱养家"，而是一条层层升级的完整传统——寄侨批养家 → 小额集资修村（祠堂、围龙屋、水井、石桥）→ 致富后大额捐建（学校、医院、路桥）→ 国难捐资救国 → 归国/回乡建设。这条史实线天然分层、可成长，与自走棋"积累→变强"的结构同构。

| 客家反哺史实层级 | 腾讯公益对应项目 | 《阿嬷的情书》呼应 | 《客批》游戏内落点 |
|---|---|---|---|
| 寄侨批养家（最基础，银信合一） | **乐捐 / 小额捐款**（99公益日"一块钱也是善意"理念） | 男主角作为番客寄钱养家 | 水客从信中收取小额银钱，攒成**桑梓值** |
| 小额集资修村、修围龙屋、打水井 | **乡村振兴计划** | — | **乡贤**用桑梓值修家园：土楼三阶段视觉（破水井→修复、断墙→翻新） |
| 致富后大额捐建学校、反哺下一代 | — | **男主角教过的学生长大致富后捐建学校**，反哺下一代 | 乡贤背景 flavor：耕读羁绊、"当年的支教青年成了乡贤" |
| 救助贫弱、独居老人、贫困儿童 | **社区救助计划** | — | 公益事件取舍（P0）/ 公益成就池（P1） |
| 国难捐资救国、归乡建设 | — | — | 公益成就池高阶事件（P1，如"国难筹赈"flavor） |

**三重咬合一句话**：① 蹭对了腾讯生态——接的是腾讯公益（乐捐/乡村振兴/社区救助），有真实项目可对应；② 合客家史实——侨民"取之南洋、用之桑梓"是百年真实传统；③ 呼应阿嬷情书——念信形式 + "学生致富后捐校反哺下一代"的真实善意闭环。

## 2. 用户与使用场景

### 2.1 目标用户

C 端玩家 / 黑客松评审。一局制单机轻策略玩家、对文化叙事与情感体验有共鸣的用户。

### 2.2 用户画像与核心诉求

- **身份**：玩过或了解自走棋的轻中度玩家；关注文化、公益、独立游戏的人群；评审。
- **使用动机**：体验一局有策略深度、又有强情感叙事的游戏。
- **核心诉求**：上手快（沿用熟悉的自走棋规则，零学习成本）+ 结局有记忆点（手势接信 + 真实侨批 + 客家话）。
- **操作习惯**：网页端，鼠标/触摸为主，结局段落用摄像头手势（含降级）。

### 2.3 典型使用场景

- 玩家打开网页，进入一局 6 关的自走棋；备战买棋、布阵、看自动战斗、攒客批；最终触发归乡结局，用手势接住一封真实侨批，听客家话朗读。
- 路演现场：评审现场试玩或观看，重点呈现结局过场的情感冲击。

## 3. 业务流程说明

### 3.1 核心业务流程（一局完整主流程）

**目标（一句话）**：**活着撑到第 6 关 + 攒够 5 封客批 → 获得归乡票 → 归乡结局。**

**赢的因果链（核心，逻辑收敛为一条线，非三个并列条件）**：
> 人守住寨子（寨子存续度 > 0）→ 寨子还在运转、有人持续收信 → 水客把客批一封封攒起来 → 客批集满 5 封 → 化成归乡票 → 归乡。
> 反之，人守不住（存续度归零）→ 寨子散了、没人攒信了 → 提前出局 → 救信结局。

**两条独立的线（V1.2 把原"家园血量"正式拆开，互不混淆）**：
- **寨子存续度（生存线）**：=还有没有人守住寨子。**初始 2**，**输一关 −1**，**归零 = 提前出局（输）**。它是客批能否攒够的**前提**，不是与客批并列的赢的条件。与客批阈值5咬合：最多输1关仍能赢，输第2关即出局。
- **家园修复值（温情线）**：=寨子修得好不好看。绑桑梓值、**只升不降、修过不回退**，驱动土楼三阶段视觉。它是攒信路上的**附加正向反馈，不参与输赢判定**。

一局 = **6 个关卡**。**客批通关阈值 = 5 封**（数值 review 见 §6.12）。每关 = 备战阶段 + 战斗阶段：

```
【局开始】寨子存续度=2，金币=初始值(见§6.0)，客批=0，桑梓值=0，家园修复值=0(土楼破败态)
   ↓
┌─────────── 单个关卡循环（重复6次）───────────┐
│ ① 备战阶段（玩家操作，有思考时间）              │
│   - 花金币在商店买棋子 / 刷新商店               │
│   - 凑3个同名同星 → 自动合成升一星              │
│   - 把棋子摆到围屋棋盘上（靠己方一侧）          │
│   - 花金币升人口（解锁更多上场棋位）            │
│   - 攒金币吃利息 / 花光抢战力（自行权衡）        │
│   ↓                                            │
│ ② 战斗阶段（自动，玩家不操作，看结果）          │
│   - 己方棋子 vs 本关遗忘军团，自动互打           │
│   - 一方棋子全部阵亡 → 战斗结束                  │
│   ↓                                            │
│ ③ 结算                                         │
│   - 胜：客批+1(固定胜场计数)；水客收信→桑梓值+；存续度不变 │
│   - 负：寨子存续度 −1；客批不增（棋子阵亡是单场内事，│
│         与家园修复值无关）                       │
│   ↓                                            │
│ ④ 修家园（温情线，只升不降）                    │
│   - 乡贤在场：消耗本关桑梓值 → 家园修复值上升     │
│       └ 土楼随修复值分三阶段变好看(详见§9.1A)    │
│   ↓                                            │
│ ⑤ （偶尔）公益事件取舍：投入 vs 不管             │
└────────────────────────────────────────────────┘
   ↓
【整局判定】
   - 撑到第6关结束 且 客批 ≥ 5
        → 客批集满，思乡能量 **具象化成一张"归乡票"**
        → 水客凭票装船启航 → 【赢的结局：风浪·手势接信】
   - 中途寨子存续度 ≤ 0（累计输到第2关）
        → 寨子守不住、提前出局 → 【输的结局：风浪·救信】
   - 撑到第6关 但 客批 < 5（信没攒够）
        → 同样进入【输的结局：救信】（人在、信不足，未能成行）
```

**"归乡票"显化说明（呼应《归途》"唯一船票"但借神不借形）**：
每封客批 = 一块"思乡碎片"；攒满 5 封，碎片汇成的能量**具象化为一张归乡票**（视觉建议：旧船票样式、盖红头船印），它才是驱动水客启航回乡的"钥匙"。**它不是新货币、不引入新数值**——只是给"客批≥5"这个通关判定穿上一件看得见、可作结局叙事扳机的衣服。归途的票要在逃杀里互相争夺（零和），我们的票是自己一封封攒出来的（积累的温柔）。

**三个量各管什么（核心判定逻辑）**：
- **武力值**：决定单场战斗胜负，赢关保住寨子存续度。
- **寨子存续度**：生存前提，归零出局；它在，才有人持续攒信。
- **客批**：唯一主线目标，集满 5 封 → 归乡票。
- **家园修复值/桑梓值**：附加温情反馈，只升不降、看着家变好，不判输赢。
- **互不相乘**：武力守寨子，客批是目标，桑梓让家变好看。

### 3.2 异常流程

- **寨子存续度归零**：累计输到第 2 关，存续度=0 → 提前出局 → 进入【输的结局：救信】。
- **撑到第6关但客批不足5**：人在、信不够 → 同样进入【输的结局：救信】。
- **手势识别失败 / 摄像头不可用**：自动降级为鼠标/触摸滑动操作结局过场（详见 §9）。
- **战斗平局 / 双方同归于尽**：判定为玩家负（按负方结算，存续度−1），避免死循环。
- **战斗超时（双方都未在时限内清场）**：设回合时限，超时按双方剩余总血量百分比判定，低者为负——防止龟缩阵容卡死。
- **第6关（末关）结束判定顺序**：先结算"客批是否≥5"，再看存续度——即末关无论胜负，只要此前累计未出局（存续度>0）且客批已≥5，即触发归乡；若末关输导致存续度恰好归零、但客批已≥5，仍判赢（已攒够信、人也撑到了最后一刻）。避免"末关输"与"撑到第6关"两个判定打架。
- **商店无可买棋子 / 金币不足**：照常进入战斗，不强制消费。

## 4. 全局通用规则

1. **页面加载 / 空数据 / 网络异常兜底**：核心为单机本地逻辑，弱网仅影响 AI 接口（对手/内容生成），AI 不可用时降级为预设规则对手 / 预置文案。结局过场资产本地化，断网仍可播放。
2. **关卡推进规则**：关卡按固定顺序推进；备战阶段有倒计时，超时自动进入战斗。
3. **按钮防抖**：购买/刷新/升人口等消费操作做重复点击防抖；金币不足时按钮置灰。
4. **手势交互校验**：结局段落实时检测摄像头权限与手势可用性，不可用即提示并切降级路径。
5. **弹窗/事件浮层**：公益事件等弹窗提供明确的"投入/不管"二选一，不可空选；选择后即时结算并关闭。

## 5. 详细功能需求（核心）

### 5.1 【核心玩法 · 自走棋循环】

**页面入口**：进入一局即开始。
**页面结构**：上方=敌方区（遗忘军团），下方=己方区（靠玩家身体一侧）；中部为围屋圆形格棋盘；侧边/底部为商店、金币、人口、客批计数、**寨子存续度（生存）**、**家园修复值条（土楼弧线）**。

#### 5.1.1 功能点1：招募 / 商店

- **前置条件**：备战阶段、金币充足。
- **用户操作**：点击商店棋子购买；点击刷新（2 金币）换一批。
- **正常流程&反馈**：扣金币，棋子进入备战席；若已有 2 个同名同星，第 3 个买入时**自动合成升星**（动效提示）。
- **异常流程**：金币不足按钮置灰；备战席满时提示无法购买。

#### 5.1.2 功能点2：升星合成

- **前置条件**：拥有 3 个**同名、同星**棋子。
- **用户操作**：自动触发（无需手动）。
- **正常流程&反馈**：3 个 N 星 → 自动合成 1 个 (N+1) 星，**角色不变、仅强度提升**（详见 §6 数值）。合成动效 + 提示。
- **异常流程**：不同角色不可合成（农夫永远不会变守卫）；不同星级不可合成。

#### 5.1.3 功能点3：布阵

- **前置条件**：备战阶段，棋子在备战席。
- **用户操作**：拖拽棋子到围屋圆形格（己方一侧），调整站位。
- **正常流程&反馈**：上场棋子数受"人口等级"限制；站位影响战斗（前侧棋子先接敌）。**仅武力棋子占用战斗人口**，公益角色见 §5.1.3A。
- **异常流程**：超过人口上限无法上场，提示需升人口。

#### 5.1.3A 功能点3+：公益后勤位（解"公益双人陷阱"，P0）

> **背景**：水客攻击=0、乡贤攻击低，若与武力棋子抢同一批上场名额，玩家会被逼进"上公益角色=战力崩=输=公益线启动不了"的死锁——这违背本作内核（公益是从胜利里长出来的，不是牺牲胜利换的）。故将公益角色挪出战斗名额。

- **设计**：棋盘下方己方战斗区之外，固定一条**后勤区（2 个固定槽位：水客槽 / 乡贤槽）**。水客、乡贤**开局即常驻后勤位**，**不占战斗人口、不进战斗、不抢战力**。
- **叙事自洽**：水客本就在后方挑担收信、乡贤本就在后方修楼——他们不该站前线打仗。前线打仗 / 后方收信修家，画面分工更清晰、更好看。
- **简化（防复杂度膨胀）**：后勤角色为**开局固定配给的功能位**，不需购买、不需升星、不占商店——就是两个常驻岗位。玩家的策略集中在战斗阵容上。
- **效果**：水客每场胜利收信（客批+1、桑梓值+）；乡贤每回合用桑梓值修家园。死锁解除——玩家专心组武力阵容打赢，公益线在后方自动运转、稳定产出。
- **感知反馈（防后勤位太静态）**：每次水客收信、乡贤修家园都配**明确的动效+音效**（挑担送信入匣的"咔"声、修楼的敲击声、土楼跨阶段时的高亮动效），让玩家虽不操作后勤、但能清楚"看到/听到"公益线在动、家在变好——后勤位是自动的，但不能是隐形的。
- **页面结构对应**：
```
┌──────────────────────────┐
│      敌方区（遗忘军团）   │
│   己方战斗区（受人口限制）│ ← 武力棋子
├──────────────────────────┤
│  后勤区  [水客槽][乡贤槽] │ ← 不占战斗人口
└──────────────────────────┘
```

#### 5.1.4 功能点4：自动战斗

- **前置条件**：备战结束。
- **用户操作**：无（自动）。
- **正常流程&反馈**：己方 vs 遗忘军团按攻击力/血量/攻速/护甲/站位自动互打，触发羁绊与技能；一方全灭即结束。
- **异常流程**：平局判玩家负。

#### 5.1.5 功能点5：结算 & 客批收集（武力→双线产出）

- **前置条件**：战斗结束。
- **用户操作**：无（自动）。
- **正常流程&反馈**：
  - 胜：寨子存续度不变；**客批固定 +1 封**（客批=纯胜场计数器，每赢一关恰好 +1，无加成、无额外，确保"阈值5=赢满5关"成立）；画面**水客挑担把信送进信匣**动效，客批计数 +1。
  - **桑梓值产出（随信收回，乡贤建设用，与客批解耦）**：信里附带番客省下的小额银钱→桑梓值，由**水客随信一并收回、但不经手使用**；桑梓值的唯一用途是**乡贤拿去建设家园**（修土楼），详见 §6.10A。公益角色加成只放大桑梓值，**不影响客批数**。
  - 负：**寨子存续度 −1**（棋子阵亡是单场战斗内的事，不影响家园修复值），客批不变。
- **异常流程**：客批达 5 即锁定**归乡票**（待第6关结束触发结局）；存续度归零即提前进入失败结局。

#### 5.1.6 功能点6：修家园（公益实体支线，桑梓值唯一去向）— P0

- **前置条件**：场上有**乡贤**，且桑梓值 > 0。
- **用户操作**：无（结算后自动结算）/ 备战时可见进度。
- **正常流程&反馈**：乡贤每回合消耗本关桑梓值，按 §6.10A 折算为**家园修复值上升**；修复值跨越 33%/67% 时，**土楼视觉升级一阶**（破败→修缮→翻新，详见 §9.1A），玩家肉眼看到"自己的家在变好看"。**只升不降，修过不回退。**
- **设计意义**：把抽象"公益值"拆成一条**看得见的物理因果链**——信里有钱→钱变桑梓值→桑梓值修楼→楼变好看。这是公益线的记忆点本体，比结局那句点睛更早、更实，是点睛之笔得以成立的地基。
- **红线（守方向A）**：桑梓值是**单向消耗的"修复进度变量"，不是第二套货币**——唯一来源=收信（被动），唯一去向=修家园，**不能交易/合成/换棋子/反向变金币**，无独立商店。本质是"家园修复值"的前置量。
- **异常流程**：无乡贤在场时桑梓值暂存不消耗（保留到乡贤上场）。

#### 5.1.7 功能点7：公益事件取舍（简版）

- **前置条件**：关卡间隙随机触发。
- **用户操作**：二选一——投入资源 / 先顾打仗。
- **正常流程&反馈**：投入 → 本关全队攻击 −20%（一回合）+ 家园修复值 +10 + 点亮一条公益 flavor（P1 计入成就池）；不管 → 保留战力。
- **异常流程**：必选其一，不可跳过。

### 5.2 【结局过场 · 手势接信】

**页面入口**：撑到第6关且客批≥5（赢）或寨子存续度归零/信不足（输）时触发。

> 以下为**初始设定值**，进开发由白糖按手感微调。升星倍率采用 demo 友好的 **每升一星 ×2**。

客家棋子有两种平行的力量，互不混淆、不相乘：
- **武力值轴**：战斗力（坦克、输出、策略），决定单场战斗胜负。
- **公益值轴**：不产生破坏力，而是**收集/守护客批、修复寨子**，决定能否撑到归乡。

> **文化依据**：客家人世代信奉、口口相传的处世古话"一命二运三风水、四积阴德、五读书……"是民间对"改变命运杠杆"的影响力排序，其中**四积阴德 > 五读书**——做好事的分量本就高于读书。游戏据此把公益角色（乡贤）的地位定在教书先生（读书）之上、风水先生（风水）之下。（注：此为客家人信奉传诵的民间古话，非客家方言本身。）

### 6.2 费用 vs 星级（关键概念，避免混淆）

- **费用 = 招募价**，棋子出厂身价，**永远不变**：农夫1费、守卫2费、教书先生3费、乡贤（公益）卡3.5档、风水先生4费、族长5费。买十个农夫每个都是1金币。
- **星级 = 合成等级**，凑 3 个相同的自动升星：3个1星→1个2星→3个2星→1个3星。**角色不变，只变强**。
- 二者独立：如"2星教书先生"=花3金币买来、已升过一级的教书先生。

### 6.4 升星数值规则（回答：2星比1星强多少）

**通用惯例：每升一星，血量与攻击力 ×2（demo 采用）。**

| 星级 | 血量 | 攻击 | 说明 |
|---|---|---|---|
| 1星农夫 | 450 | 35 | 基础 |
| 2星农夫 | 900 | 70 | 各×2 |
| 3星农夫 | 1800 | 140 | 再×2 |

- 攻速、护甲、距离不随星级变（仅血/攻翻倍）。
- 升星=该角色自己变强，**绝不变成别的角色**。

### 6.5 攻击掉血逻辑（升星如何影响掉血）

**单次攻击伤害公式：**
```
实际伤害 = 攻击力 × (100 / (100 + 目标护甲))
```
- 1星农夫(攻35)打护甲10的敌人：35×(100/110)=31.8 实伤
- 2星农夫(攻70)打同一敌人：70×(100/110)=63.6 实伤 → **攻击翻倍则掉血翻倍**，自动联动，无需另设规则。
- 攻速决定出手频率：攻速0.6=每1.67秒出手一次。

### 6.6 2星农夫 vs 2费守卫（升星 vs 高费的差异）

| 对比 | 2星农夫(1费×3合成) | 1星围屋守卫(2费直接买) |
|---|---|---|
| 血量 | 900 | 950 |
| 攻击 | 70 | 40 |
| 护甲 | 5 | 25 |
| 定位 | 升级后的廉价高输出，脆 | 天生坦克，耐揍 |

**设计哲学**：低费棋子靠"凑3个升星"获得强度跃升、偏输出但脆；高费棋子靠"天生属性好"、偏坦克稳健。玩家策略=赌运气凑低费升星 vs 花钱买高费稳坦克。高费棋子商店出现概率低、又贵，前期钱少只能先用便宜棋子过渡——经济与概率逼出策略深度。

### 6.8 升星链定稿（选项甲：严格遵守自走棋规则）

- **每个角色只升自己的星**：3个1星乡贤→1个2星乡贤→3星乡贤，乡贤始终是乡贤。
- 教书先生、乡贤、风水先生是**三个独立棋子**，各买各升，**不跨角色进化**。
- **公益线如何承载**：乡贤本身就是公益角色（修寨子、多收信、强化文化羁绊），升星=公益能力变强。
- "返乡青年成长（志愿者→支教→乡贤）"的叙事保留为**乡贤的背景flavor文案**（升星时配一句"当年的支教青年，如今成了乡贤"），不做成跨角色合成系统。叙事在，系统不复杂。

|---|---|---|---|

### 6.10 两条线的数值逻辑（生存线 + 温情线，互不混淆）

**A. 寨子存续度（生存线，判输赢）**
```
初始 = 3 点
每输一关 −1（棋子被遗忘军团全灭=本关负）
归零（累计输到第2关）→ 提前出局 → 失败结局
```
- 它衡量"还有没有人守住寨子"，是客批能否继续攒的前提。
- 赢关不增（封顶3）；只减不增，制造"还能输几关"的生存张力。

**B. 家园修复值（温情线，不判输赢）**
```
初始 = 0（土楼破败态）
仅靠桑梓值修复上升，只升不降、修过不回退
驱动土楼三阶段视觉（详见 §9.1A）
```
- 它衡量"寨子修得好不好看"，是攒信路上的附加正向反馈。
- **战斗失败不会让它下降**（棋子阵亡是单场内的事，与家园修复无关）——这是它"温情、纯粹向上"的关键。

### 6.10A 桑梓值机制（驱动家园修复值）与命名方案

**机制（这条链是公益线的灵魂，全栈照此实现）：**
```
水客收信(被动·赢关时) → 客批 +1(固定，纯胜场计数，详见§5.1.5)
                      → 信里附带桑梓值随信一并收回(水客只运不用)
公益角色加成 → 放大桑梓值(不碰客批)
       ↓ (桑梓值唯一去向)
乡贤(建设者) → 每回合把桑梓值用于修家园 → 家园修复值上升(只升不降) → 触发土楼视觉升阶
```
- **客批与桑梓值彻底解耦**：客批=胜场数(每关+1封，封顶即通关阈值)，桑梓值=修家园资源(可被各种加成放大)。两者来源同为"赢关"，但口径独立，互不换算。
- **来源唯一**：仅来自水客收信（赢关时被动产出），玩家无需额外操作。
- **去向唯一**：仅供乡贤修家园（转家园修复值）。
- **红线**：不可交易、不可合成、不可换棋子、不可反向变金币、无独立商店。它是"家园修复值"的前置量，**不是第二套货币**——守方向A。
- **建议折算（初始值，白糖手感微调）**：设计目标是**正常推进下约 6 关内走完"破败→修缮→翻新"完整弧线**——即每赢一关约推进家园修复值 +15~17%（3 关左右跨一档）。桑梓值不足时按实有量修复。

**命名方案（明天与白糖/Murphy 敲定）：**
| 方案 | 词 | 含义 / 优点 | 缺点 / 风险 |
|---|---|---|---|
| **主推** | **桑梓值** | 取典"取之南洋、用之桑梓"，精准命中客家侨民反哺家乡的史实与情感，最美、最有文化分量 | 典故偏雅，部分评委/玩家可能不能"一眼 get"其=修家园资源，需 UI 辅助说明 |
| Plan B | **乡心** | 一个词同锁"思乡碎片"+"反哺善心"两层灵魂，直白易懂，明确不像钱（规避"第二套货币"误读） | 较通俗，文化辨识度弱于桑梓 |
| Plan C | **批银** | 直接用侨批术语，"批=信、银=钱"，银信合一，最贴史实本体 | **太像货币**，容易被误读成第二套钱，与方向A红线观感冲突 |

> 倾向：主名用**桑梓值**（文化分量最高），UI 上加一句注释/图标辅助理解；若现场测试发现评委 get 不到，降级到 Plan B 乡心。最终由团队拍板。

### 6.11 关卡结构（定稿）

- 一局 **6 关**，对应结局最多 5 封玩家攒得的真实客批（另备 AI 数字客批作沿途 flavor）。
- **客批通关阈值 = 5 封**：6 关里赢够 5 关即达标，归乡票触发（依据见 §6.12）。
- 难度：1–2 关教学（遗忘弱）；3–4 关上强度；5–6 关 AI 动态难度介入。
- **赢 = 撑到第6关（存续度>0）且客批≥5；输 = 存续度归零提前出局，或撑到第6关但客批<5。**

### 6.12 数值与玩法 Review（难度合理性论证 → 阈值/关卡定稿依据）

> 结论先行：**关卡=6、客批阈值=5、寨子存续度=2（输-1归零出局）、家园修复值绑桑梓值只升不降**这组数值，难度曲线健康、张力适中、路演时长可控（约 10–12 分钟一局）。以下为推演依据，明天可直接和白糖对数。

**① 生存张力与目标张力咬合（存续度=2 × 阈值=5）**：两条线指向同一个要求——**最多只能输 1 关**。输第 2 关时存续度归零、立即出局进救信结局；不会出现"已注定输还被迫打完剩余关"的尴尬。玩家心智极简：**输 2 关 = 结束**。

**② 三种结局路径**：
- 全胜(6胜0负)：客批6≥5 ✓、存续度2 ✓ → 赢（轻松，第5关即达标）。
- 5胜1负：客批5≥5 ✓、存续度1 ✓ → 赢（达标线，输1关的紧张感）。
- 输到第2关：存续度0 → 立即出局 → 输（救信结局），干净利落。

**③ 家园修复值/土楼弧线匹配**：按"每赢一关约 +17%"折算，全胜路径下第2关进"修缮"、第4关进"翻新"，**6 关内能完整看到"破败→修缮→翻新"三阶段**，破败态在开局看得到，温情弧线完整且只升不降。

**④ 难度结论与建议**：
- 曲线"生存压力 + 目标压力"双线叠加，张力适中，**保留**。
- **微调旋钮（交白糖手感）**：嫌太难（输1关就慌）→ 客批阈值降到 4（允许输2关，但需同步把存续度升到 3 才咬合）；嫌太易 → 敌方单位数+1 或后期关卡系数加重。**关键纪律：存续度与客批阈值必须咬合——"可容忍输 N 关"应同时满足 存续度>N 且 (6−N)≥阈值。当前 N=1：存续度2、阈值5,咬合。**
- **为何不再用"家园血量清零判输"**：把生存（存续度）与温情（修复值）彻底拆开后，土楼修复成为纯粹向上的成就线、不会因战败回退，温情弧线不被达标压力污染；生存张力清晰落在"还能输几关"上，玩家心智模型只需记"别输超过1关 + 攒够5封信"。

**⑤ 阵容取舍（V1.3 后勤位调整后）**：水客/乡贤进后勤位、不占战斗名额，因此"做公益"不再以牺牲战力为代价（修掉了"公益双人陷阱"）。策略深度转移到武力阵容内部：守卫/教书/风水/族长之间的费用-星级-羁绊取舍，以及"贪经济堆农夫 vs 稳战力"的权衡。公益线在后方稳定运转，与武力线并行不抢资源——这正贴合史实"公益是从胜利里长出来的，不是牺牲胜利去换的"。

### 6.13 AI 落点设计（AI 自走棋的"AI"含量，腾讯云黑客松考点）

> 本作是 **AI 自走棋**，需让评委看到 AI 不是噱头而是嵌进玩法/文化。以下按性价比排序，P0 必做 1 个，P1 候选 2 个附工作量，供明天与 Murphy 讨论取舍。

**【P0·必做】AI 落点一：AI 生成"数字客批"**
- **做什么**：玩家沿途攒的客批，信件内容由 AI **基于真实馆藏侨批的语气/句式 + 本局战况（打了几关、家修到几成、救了哪些信）动态生成几句"番客口吻的家书"**，每局不同。
- **为什么出彩**：一次命中"AI × 文化 × 公益"三题——AI 生成、客家家书文化、侨民反哺情感。且每局信不同 = 重玩价值。
- **红线（合规）**：**结局正式朗读的那封信，仍用真实馆藏侨批原文（见 §11.4，版权安全）**；AI 生成的家书仅作"沿途攒的信"的 flavor 文本展示，**不作客家话朗读主体**，避免 AI 杜撰文献的风险。
- **工作量**：中。一个 prompt 模板 + 战况变量注入 + 调用接口，弱网降级为预置文案池。

**【P1·候选，附工作量，明天讨论】AI 落点二：AI 难度旁白官**
- **做什么**：把已有的"AI 对手/动态难度"叙事化——AI 按玩家阵容实时调节遗忘军团强度，并生成一句"历史阻力"口吻的旁白（如公益线弱时，迁海碑冷语"过番容易返唐难"）。
- **价值/成本**：把已有 AI 落点包装出叙事感，几乎不增加新系统，**工作量小**；风险是旁白文案需控调性，建议预置+AI 混用。

**【P1·候选，附工作量，明天讨论】AI 落点三：AI 生成公益/文化事件文案**
- **做什么**：§5.1.7 公益事件的随机文案，由 AI 基于客家反哺史实池（§11.7 五层级）生成，每局不同（修祠堂/集资建校/捐建卫生院…）。
- **价值/成本**：增强公益线纵深与重玩感，**工作量中**（需事实约束 prompt 防跑偏）；可与 P1"公益成就池"合并实现，性价比更高。

> **讨论话口（给明天）**：P0 先锁"数字客批"（最出彩、扣三题）。落点二工作量最小、建议优先加；落点三可与公益成就池打包评估。三个 AI 落点叠加后，"AI 自走棋"的 AI 含量在叙事层（旁白/家书）+ 机制层（动态难度）+ 文化层（史实事件生成）三处可见，评委一眼能看到 AI 是嵌进作品而非贴皮。具体做几个，看 Murphy 的 3-4 天工期余量定。

## 7. 遗忘军团：敌人设计（历史阻力拟人化）

> 设计原则：**物变人 / 人变物**，玩家一眼看懂"这是阻挡客家人回乡、造成遗忘的东西"。每仗都是与"阻挡归乡的历史"作战。以下为美术设定依据。

| 敌人（拟人化） | 历史成因（阻挡返乡） | 形象方向 |
|---|---|---|
| **迁海碑（禁海碑）** | 明清海禁/迁海令：官府禁止私自出海，并捉拿偷渡返乡者治罪。客家人一旦过番，便不敢私自回乡（回则抄家正法）——人被一道海禁隔在故土之外，"过番容易返唐难" | 长脚的官府界碑，盖满朱红官印，碑文发光镇压 |
| **路引关吏** | 关卡路票管制：官府在水陆要道设卡，返乡须持官方路引，无凭证当场扣押。出外开荒者手续不全，无法折返原籍 | 戴官帽、举"路引"令牌的差役，似关卡门神，拦路查票 |
| **猪仔契** | 契约劳工（卖猪仔）：底层客家人被卖为南洋契约华工，签数年卖身契，矿场/橡胶园严控人身，合约未满无自由，熬完多身无分文连船票都凑不齐 | 活过来的卖身契卷轴，化锁链缠人，链上是矿场/橡胶园符号 |
| **饿虎山** | 内陆群山阻隔+故土无生路：闽粤赣山区八山一水一分田，武夷/南岭包围，无路无产业，空手回乡只会挨饿 | 有獠牙的拟人化山岭，身覆枯田，挡在归途 |
| **红头船（天价归船）** | 返乡成本极高：红头船船票+数十天航程食宿+预留安家银两，普通人打拼十几年攒不下路费，终身只能靠侨批寄钱、不见故土 | 拟人化红头船，船身贴满天价船票/银元符号，横在归途索买路钱，付不起就过不去 |
| **械斗火** | 土客大械斗：粤桂土客因争地爆发大规模械斗，外出避难者不敢回乡（怕报复排挤），家乡无立足之地 | 一团持械、有手脚的火，代表回乡遭报复 |

## 8. 结局设计（电影彩蛋式意象，赢/输双版）

### 8.1 设计定位

结局是**电影彩蛋式的意象之美**，不是"对抗抢救"的紧张戏。两版共用同一套过场资产（风浪+子弹时间+手势接信+真实侨批+客家话念信），仅叙事框与"信的来源方"不同，技术上几乎零额外成本。

### 8.2 赢的结局（归乡）

当客批集满 5 封，**这些"思乡碎片"汇成的能量具象化为一张归乡票**——水客凭这张票，把你赚来的 N 封信一封封装船送回家乡。归途中风浪掀起，信件散落空中，你伸手一封封接住、打开。意境温柔——如《西游记》最后一难在通天河，苦尽之后的最后一程，你亲手护住了这些牵挂。
> 收尾字幕："你让 N 个客家人的牵挂，回了家。"

### 8.3 输的结局（救信，温情扣题版）

寨子终究没能守住，遗忘的浪头漫了上来。但当满载牵挂的信被卷进风浪里，你没有转身——你伸手，从浪里一封封救起它们。你救起的这一封，依然漂洋过海，抵达了它想去的地方。失败者也有尊严的收束：你没赢下所有，但你至少护住了一份牵挂。
> 收尾字幕："你没能赢下所有，但你至少让一个客家人的牵挂，回了家。"
> 与赢版结构对称（"N 个" vs "至少一个"），都落到温情 + 扣题；接同一句科普 + 腾讯公益："善意未必每次都圆满，但它一直在抵达远方——就像今天，这件事仍有人在做。"

### 8.4 致敬边界（合规）

致敬《阿嬷的情书》的是**朗读形式、信件吹散/落水的镜头语言、音义双关手法、思念乡愁的情绪基调**；**不使用其原创信件文字**。结局信件一律采用**真实馆藏侨批原文**（标注出处，见 §11.4），客家话朗读——真实文献分量远超虚构文案，且无版权风险。

## 9. UI与交互说明

### 9.1 棋盘布局

- **去掉内外圈**。整个围屋内部 = 棋盘，方格改**圆形格**，呼应圆形土楼。
- **上下分敌我**：靠玩家身体一侧（下方）= 己方棋子；远离一侧（上方）= 敌军。与下棋认知一致，达成共识性。
- 视觉用**圆形土楼**（福建龙岩永定式）——记忆点远强于方楼，一眼识别"客家"。

### 9.1A 土楼三阶段视觉态（公益线记忆点本体，P0）

家园修复值不是一根普通血条——它**绑定围屋棋盘本身的视觉状态，只升不降**，让玩家肉眼看到"自己的家在桑梓值修复下一步步变好看"。这是把"修复"做成情绪、而非进度条的关键。修复顺序按客家人重建家园的真实逻辑：先保命（水、门、住人），再恢复生产生活（晒谷、祠堂、灯火）。

**给美术/前端：只需三套主图（破败/修缮/翻新），跨节点的"水井出水""牌匾挂起"做成切换瞬间的小动效叠加即可，不必画第四第五套——省工期。**

| 修复值 | 阶段 | 这一阶修好了哪里（数值对应的具体部位） |
|---|---|---|
| 0–33% | **破败**（初始态） | 夯土墙开裂剥落、屋瓦塌陷露窟窿、**水井干涸长草**、大门歪斜、晒谷场荒芜、全屋无灯火、色调灰冷。呼应"离乡后家园荒废、被遗忘吞噬" |
| 跨 33% 触发 | 破败→修缮 动效 | **水井先出水**（井台修好、打上水）+ 歪斜大门扶正修好（先保"有水喝、能进门"） |
| 34–66% | **修缮中** | 累加：开裂夯土外墙补砌平整、塌陷屋瓦补全、晒谷场清理出来开始晒谷、墙头透出零星暖光 |
| 跨 67% 触发 | 修缮→翻新 动效 | **祠堂/堂屋牌匾重新挂起**（呼应宗族羁绊），色调由灰冷转暖 |
| 67–100% | **翻新** | 全屋焕然：夯土墙金黄饱满、屋瓦齐整、**满屋灯火通明**、晒谷场谷物丰盈、炊烟升起、天井恢复生机。反哺成效的视觉高潮，强化"用之桑梓" |

- **触发**：家园修复值跨越 33%/67% 时，土楼贴图/图层**升一阶**（只升不降——修过的部位不会因战败回退，这是温情弧线纯粹向上的关键）。
- **意义**：玩家在玩的过程中就持续获得"我在把家修好"的实体感受，**为结局"你让 N 个客家人的牵挂回了家"提供情感地基**——没有前面看得见的修复，结局那句升华会立在虚处。
- **工期降级预案（若 3-4 天吃紧）**：三阶段可砍成两阶段（破败/翻新），保留开局破败与终局翻新的强对比，记忆点仍在。


### 9.2 结局手势交互分镜（写给前端）

1. **俯角**：归乡票达成，水客把整局客批装船，扬帆远航离岸（输：敌军船载抢走的信）。
2. **风浪**：海上掀起大风大浪，船倾，所有信被卷上天空；镜头俯→仰，漫天客批如雪片飘散（灵感：巴塞罗那诗歌轰炸）。
3. **子弹时间**：玩家左右滑动手势 → 信件下落变慢（黑客帝国感）。唯一交互核心，必须丝滑。
4. **画布旋转**：画面如画布缓缓旋转，中央信件高亮，玩家滑动切换、选定一封。
5. **落水致敬**：部分未选中的信飘落入海消失（致敬《阿嬷的情书》遗失意象，亦是史实）。
6. **握拳→张开**：握拳抓住高亮信 → 张开手掌 → 信封翻开。
7. **真实侨批**：呈现真实客批原文（繁体录文）+ 现代释文对照，配客家话朗读，背景可选客家山歌。
8. **可重复**：可继续开别的信，每封真实侨批+客家话朗读，预设约10封。
9. **收尾字幕**："你让 N 个客家人的牵挂回了家。" + 科普：侨批2013年入选《世界记忆名录》。

### 9.3 文案/动效规范

- 棋子立绘：客家宗族角色 + 6 个遗忘敌人，风格统一。
- 客批信件：真实馆藏侨批，繁体录文 + 释文对照排版。
- 朗读：客家话录音（多封）。

### 11.7 客家侨民反哺史实脉络（公益线史实依据）

> 用于 §1.6 四列咬合。客家"过番赚银、回乡兴家兴国"是贯穿百年的完整传统，侨批只是这条家国脉络里最基础的一环。脉络层层升级：

1. **侨批养家（清末–1980 年代）**：银信合一，南洋客家人省吃俭用每月汇款，支撑老家父母妻儿吃饭、看病、婚嫁；近代粤东、闽西乡村大半家庭靠侨汇维持生计。批信常附"乡众公用钱"，全村侨胞凑钱修祠堂/围龙屋、铺村道、建石桥、打水井。
2. **致富后大额捐建公益**：兴办学堂（崇文重教第一要务，梅州几乎所有老中学/乡镇小学为南洋客侨捐建，如乐育中学）；修建医院卫生院（解决山区缺医少药）；交通水利市政基建（华侨集资建梅江桥；张榕轩兄弟斥巨资建成中国第一条华侨民营铁路——潮汕铁路）。
3. **国难救国（辛亥革命、抗战）**：客家侨领捐巨款资助革命；东南亚客家总会组织"筹赈祖国难民委员会"，矿工小贩月捐工钱、输送药品军火、青年回国参军。
